## 三、部署步骤

```bash
# 1. 确认代码为最新 main 分支
git checkout main
git pull

# 2. 构建验证（已通过）
npm run build

# 3. 推送代码
git push origin main

# 4. 在 Vercel 中部署
#    - 导入仓库 https://github.com/你的用户名/wisescan
#    - Framework: Vite
#    - Build: npm run build
#    - Output: dist
#    - 配置环境变量（见上表）
#    - 绑定域名 wisescan.xyz

# 5. 部署后验证
#    curl https://wisescan.xyz/api/health
```

---

## 四、需要你手动确认的项

1. **改 ADMIN_PASSWORD** — 在 Vercel 环境变量里设置
2. **Supabase RLS 策略** — 进 Supabase 控制台 → Authentication → Policies，确认 `projects` 和 `risk_reports` 表有行级安全策略
3. **域名解析** — `wisescan.xyz` 的 DNS 是否已指向 Vercel
4. **Vercel 环境变量** — 把 `.env` + `.env.local` 里的所有变量配到 Vercel 生产环境
