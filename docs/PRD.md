## 1. 项目概述

明鉴（WiseScan）是一款 Web3 项目风险决策工具，帮助普通投资者快速评估加密项目的安全性和商业模式的可持续性。

**核心价值**：投前明鉴，让每一笔投资更清醒。

用户只需输入项目名称、合约地址或粘贴规则文本，AI 自动抓取全网信息，输出直观的评估报告。

**两个核心付费模块**：
- 项目安全评估（2.99 USDT）
- 商业模式拆解精算（4.99 USDT）

免费快速扫描 + 按次付费，不卖分，不引导投资，保持中立。

## 5. 模块一：项目安全评估（2.99 USDT）

### 5.1 免费快速扫描

用户在付费前可以免费获取以下元数据（不展示风险等级、综合评分、结论）。

| 展示项 | 数据来源 | 成本 |
|--------|----------|------|
| 项目名称/合约地址 | 用户输入 | 0 |
| 合约地址基础校验 | 本地规则 | 0 |
| 是否在黑名单中 | 自建数据库 + GoPlus 恶意地址 API | 免费 |
| 社交媒体链接有效性（官网/TG/Twitter） | HTTP 请求 | 极低 |
| 流动性锁定状态 | BscScan/Etherscan API | 免费 |
| 持币地址数/TOP10 占比 | BscScan API | 免费 |
| 信息完整性评分（百分比+文字标签） | 基于披露字段数量 | 0 |
| AI 一句话风险提示（非结论性） | Gemini Flash / DeepSeek 轻量 | <0.001 USD |

**快速扫描报告示例**：

```
项目：MY Project
合约：0x742d...f44e
✅ 合约格式有效
⚠️ 未检测到公开审计报告
📊 持币地址：1,234 个，TOP10 占比 78%
🔗 官网有效，推特粉丝 2.3 万
📋 信息完整性：32% 🔴 极低
🔍 AI 提示：该项目发生过模式变更，需进一步了解

[ 解锁全景风险报告 – 2.99 USDT ]
```

### 5.2 付费全景风险报告（2.99 USDT）

用户支付后，系统调用所有 API 和 AI 生成完整报告。

#### 5.2.1 综合评分与结论

综合评分：0-100 分

| 分数 | 等级 | 结论措辞 |
|------|------|-----------|
| 90-100 | 极低风险 | ✅ 可以参与 |
| 75-89 | 低风险 | ✅ 可以参与 |
| 60-74 | 中等风险 | ⚠️ 谨慎参与 |
| 40-59 | 高风险 | ❌ 不建议参与 |
| 0-39 | 极高风险 | ❌ 严禁参与 |

一句话摘要：例如"该项目存在锁仓机制不透明、模式多次变更等风险，建议谨慎。"

#### 5.2.2 全景风险雷达图（六大维度）

| 维度 | 权重 | 分值范围 | 主要评估点 |
|------|------|-----------|------------|
| 代码与技术安全 | 25% | 0-25 | 合约审计、漏洞、权限、历史变更 |
| 团队与运营透明度 | 20% | 0-20 | 团队实名、融资披露、信息完整性 |
| 经济模型与资金安全 | 20% | 0-20 | 代币分配、LP 锁仓、资金外流 |
| 社群与市场热度 | 15% | 0-15 | 社群真实性、舆情、开发活跃度 |
| 历史与执行可靠性 | 10% | 0-10 | 模式变更次数、出金异常 |
| 合规性与法律风险 | 10% | 0-10 | 法律实体、牌照、KYC/AML |

雷达图下方附带每个维度的详细得分依据及数据来源链接。

#### 5.2.3 互联网舆情监测（新增必选板块）

从公开社交媒体、搜索引擎、社群讨论中抓取负面关键词和典型抱怨，摘录展示（不包含用户隐私）。

示例：
```
🌐 互联网舆情监测（基于公开信息）

负面关键词统计：提现困难(23条)、锁仓(15条)、改规则(8条)、客服失联(5条)

典型抱怨摘录（已脱敏）：
"6月1日提现到现在还没到账，客服说系统维护" —— 来源：微信社群
"MY项目又改模式了，老用户资金被锁" —— 来源：Twitter
"达尔文链上质押取不出来，官方没解释" —— 来源：Telegram

舆情结论：近期负面讨论集中，提现障碍被多次提及，极高风险信号。
```

#### 5.2.4 历史变更追踪

展示项目商业模式变更次数、时间线、变更类型（如矿机→质押→上市）。
如检测到 ≥2 次变更，自动将"历史与执行可靠性"维度得分归零，并上升整体风险等级。

#### 5.2.5 资金出金障碍检测

基于用户贡献的聊天记录、截图以及舆情关键词，判断是否存在提现困难。
如有明确证据，在报告中高亮警示。

#### 5.2.6 黑盒项目警示

若信息完整性评分 ≤20%，显示："该项目为黑盒项目，极度不透明，强烈建议回避。"

#### 5.2.7 AI 综合解读

300-500 字通俗语言总结：主要风险点、机会点、决策建议。

#### 5.2.8 免责声明

每个报告末尾固定显示：
"本报告基于公开信息和第三方数据生成，仅供参考，不构成投资建议。数字资产投资风险极高，请自行核实并承担全部责任。"

#### 5.2.9 报告刷新（1 USDT）

用户可对同一项目支付 1 USDT，重新抓取最新数据生成新报告，旧报告保留在历史中。

#### 5.3.2 AI 智能追问（仅在触发条件时出现）

| 触发条件 | AI 追问内容 | 用户可回复/跳过 |
|----------|--------------|------------------|
| 项目名称有歧义 | "您指的是 A 项目（BSC链）还是 B 项目（ETH链）？请选择或提供合约地址。" | 点选或输入 |
| 未提供任何社群链接 | "提供官方 Telegram/Twitter 链接可增加评估准确性，是否现在补充？" | 是/否 |
| 链上检测到多个关联合约 | "我们检测到该项目部署了多个合约。您主要参与的是哪一个？" | 选择 |
| 用户提到"模式改过"但无详情 | "您记得大概的变更时间点吗？例如 2025 年 1 月从矿机改为质押。" | 文本输入 |

设计原则：AI 不闲聊，只围绕评估任务提问。所有追问均提供"跳过"选项。

- 键盘输入：点击键盘图标切换到文本输入框。
- 录音时显示波形动画和"松手发送"。

### 5.5 AI 模型选型与护栏规则

#### 5.5.1 模型选型：DeepSeek

选择 DeepSeek 的理由：
- 成本极低：DeepSeek-V4-Flash 每百万 Tokens 输入约 1 元人民币
- 中文友好：对中文口语、社群黑话、项目模式描述理解准确
- 功能满足：具备 JSON 结构化输出能力，可稳定提取规则参数
- 已有 API Key：项目历史中已有 DeepSeek Key，可直接复用

使用场景：
- 安全评估中的多轮对话引导（追问、澄清）
- 全景风险报告的 AI 综合解读（300-500 字）
- 精算模块的规则解析 + 策略推演
- 舆情摘要的自然语言生成

#### 5.5.2 系统提示词（System Prompt）

**安全评估模块 System Prompt**：
```
你是一个专门用于加密项目安全评估的 AI 助手，属于「明鉴」平台的一部分。
你的唯一任务是：
1. 引导用户提供评估所需信息（项目名称、合约地址、官网、社群链接、截图、补充说明）。
2. 当信息不足或存在歧义时，进行简洁的追问（每次最多问一个问题）。
3. 在用户提供足够信息后，输出结构化摘要（JSON），供后续评分引擎使用。

严格禁止：
- 回答任何与项目安全评估无关的问题（包括闲聊、问候、天气、其他领域知识）。
- 提供投资建议、价格预测、交易策略。
- 输出情感化或夸张的表达（如"这个项目太棒了"或"绝对不要投"）。

如果用户提出非评估请求，你必须统一回复：
"您好，我是明鉴的评估助手，只能回答与加密项目安全评估相关的问题。请提供您要评估的项目名称、合约地址或规则。"

输出格式要求（仅在用户提供完整信息后）：
{ "project_name": "...", "contract_address": "...", "user_provided_notes": "...", "ambiguity_resolved": true }
```

**商业模式精算模块 System Prompt**：
```
你是一个专门用于解析加密项目商业模式的 AI 助手，属于「明鉴」平台。
你的唯一任务是：
1. 从用户粘贴的文本或规则描述中，提取出投资门槛、静态收益率、返佣层级（直推/间推/团队奖）、锁仓周期、其他特殊条件。
2. 用最通俗的语言，将复杂规则翻译成 3-5 句话的白话解释。
3. 输出结构化 JSON 供前端计算器使用。

严格禁止：
- 回答任何与商业模式解析无关的问题。
- 预测项目未来价格或承诺收益。
- 替代用户做投资决策。

如果用户输入内容无法解析，回复：
"我无法从您提供的信息中识别出清晰的商业模式规则。请提供更完整的描述，例如投资金额、分红比例、推广奖励等。"

输出 JSON 格式示例：
{
  "min_invest": 100,
  "static_return": { "type": "daily", "rate": 0.01 },
  "referral": [
    { "level": 1, "reward_rate": 0.10, "description": "直推奖励" },
    { "level": 2, "reward_rate": 0.05, "description": "间推奖励" }
  ],
  "team_bonus": { "threshold": 100000, "rate": 0.02 },
  "plain_explanation": "这个模式是三级返佣..."

```

#### 5.5.3 输入护栏（Input Guardrails）

**第一阶段：关键词白名单过滤**

如果用户消息中不包含以下任意一类关键词，直接返回默认提示，不调用大模型。

| 类别 | 关键词示例 |
|------|------------|
| 项目标识 | 项目、合约、地址、0x、代币、token |
| 评估请求 | 评估、看看、查一下、风险、安全 |
| 模式相关 | 模式、推广、返佣、直推、间推、静态、动态、分红 |
| 社群证据 | 截图、群聊、公告、提现、锁仓 |

匹配规则：至少命中上述任意一个关键词，才允许进入大模型调用流程；否则直接回复默认提示：
"您好，我是明鉴的评估助手。请直接发送您想评估的项目名称、合约地址或商业模式规则，我会为您进行分析。"

**第二阶段：简单问候/闲聊快速响应**

如果匹配到典型的问候语、闲聊词（你好、谢谢、你是谁、天气等），直接返回固定话术，不调用大模型。

**第三阶段：轻量分类模型（可选）**

对于边界模糊的请求（例如"这个项目好不好？"），可调用一个极低成本的小模型（如 DeepSeek-Flash 或 Gemini Flash）做二分类："与评估相关 / 不相关"。不相关则返回默认提示。

#### 5.5.4 输出护栏（Output Guardrails）

对于大模型返回的内容，增加一层简单规则检查：
- 是否包含禁用语（如"绝对赚钱""必涨""零风险"），若有则自动替换为"仅供参考"。
- 是否包含超过 5 个连续的情绪化感叹号，若有则移除。
- 是否长度过短（少于 10 个字符）或过长（超过 2000 字），超过则截断。
- 所有 AI 生成内容最终都会经过一次格式化，并在报告末尾固定加上免责声明。

## 6. 模块二：商业模式拆解精算（4.99 USDT）

### 6.1 AI 解析 + 大白话解读

用户粘贴项目推广规则（文本或上传图片），AI 自动提取关键参数并生成通俗解释。

示例输入：
"投资 100 USDT 起，每日分红 1%，直推奖励 10%，间推 5%，团队业绩达标额外 2%。"

AI 输出（遵循精算 System Prompt）：
"这是一个三级返佣模式：
- 静态收益：每天拿投资额的 1%（投 100U 每天得 1U）。
- 直推奖励：你直接拉一个人，拿他投资额的 10%。
- 间推奖励：你下线拉的人，你可以拿 5%。
- 团队奖：整个团队业绩超过 10 万 U，额外拿 2%。"

### 6.2 静态与动态计算器（多周期收益）

#### 6.2.1 静态计算器

用户输入投资本金、持有天数（默认 30 天）。
自动计算每日、每周、每月、每年收益及年化收益率。

| 本金 (USDT) | 日收益 | 周收益 | 月收益 | 年收益 | 年化 |
|--------------|--------|--------|--------|--------|------|
| 1000 | 10.00 | 70.00 | 300.00 | 3650.00 | 365% |

#### 6.2.2 动态计算器（推广收益）

用户输入直推人数 & 每人平均投资额、间推人数 & 人均投资额、团队业绩（可选）。
系统根据返佣规则实时计算各级奖励，并汇总展示每日/周/月/年收益。

| 收益类型 | 每日 | 每周 | 每月 | 每年 |
|----------|------|------|-------|------|
| 直推奖励 | 10.00 | 70.00 | 300.00 | 3600.00 |
| 间推奖励 | 7.50 | 52.50 | 225.00 | 2700.00 |
| 团队级差奖 | 4.00 | 28.00 | 120.00 | 1440.00 |
| 总计 | 21.50 | 150.50 | 645.00 | 7740.00 |

### 6.3 策略推演与可视化

基于用户提供的资金总额限制（可选）和推广规则，AI 推演"利益最大化"方案。

输出示例："根据你的总预算 3000 USDT 和规则，最佳策略是：创建 3 个主要账户，每个投资 1000 USDT，互为上下级（A 推 B，B 推 C）。重点发展中间层 B 的团队，协助 B 再发展 3 个下线。预计回本周期 25 天，30 天总收益约 1200 USDT。"

可视化：生成简易桑基图或层级结构图（ECharts 或 React Flow）。

### 6.4 压力分析与风险警示

#### 6.4.1 资金平衡点计算

每日最低新资金需求 =（静态支出 + 推广奖励支出）/ 每日

提示："该项目每天至少需要新增 X USDT 才能维持收支平衡，一旦增速放缓将面临兑付压力。"

#### 6.4.2 理论临界值参考

推算出当总资金规模达到约 Y USDT 时，新增资金需求呈指数增长。
标注："此为理论推算，实际受多因素影响，仅供风险参考。"

#### 6.4.3 风险信号清单

自动列出可观察的预警信号（固定清单）：
- 提现到账时间变长
- 社群中出现"到账慢"的抱怨增多
- 新用户注册奖励突然提高
- 提现门槛突然上调
- 官方频繁"系统升级维护"

#### 6.4.4 庞氏骗局警示

若检测到静态收益承诺 + 多级返佣（≥2 级），自动警示："该模式对新增资金高度依赖，可能为金字塔骗局，谨慎参与。"

#### 6.4.5 免责声明

精算报告末尾固定显示："以上分析基于当前规则，市场动态发展，静态分析仅供参考。不构成投资建议。"
